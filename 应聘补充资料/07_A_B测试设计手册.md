# 电商 A/B 测试设计手册

> 数据驱动不是喊口号，而是靠一套可复用的实验体系。本手册覆盖电商场景下7类核心A/B测试的设计方法、样本量计算、判定标准及常见误区。

---

## 一、A/B测试基本原则

### 1.1 核心逻辑

```
假设 → 分组 → 运行 → 收集数据 → 统计检验 → 决策
```

### 1.2 必须遵守的5条铁律

| 铁律 | 说明 | 反面案例 |
|------|------|----------|
| **单一变量** | 每次只测1个变量 | 同时改主图+标题+价格，不知道哪个有效 |
| **随机分组** | 用户随机分配到A/B组 | 按地区分组（一线城市A组，二线B组），结果不可比 |
| **样本充足** | 每组至少达到最小样本量 | 测了3天就下结论，数据量不够 |
| **运行周期完整** | 至少覆盖1个完整周期（7天/14天） | 工作日测A，周末测B，流量质量不同 |
| **统计显著** | p值<0.05才认定有效 | 凭感觉说"B组看起来更好" |

## 二、7类核心测试方案

### 2.1 主图测试

**测试目标**：找到点击率（CTR）最高的主图

| 维度 | 设置 |
|------|------|
| **变量** | 主图（A：产品特写；B：场景穿搭；C：工艺过程） |
| **指标** | 点击率（CTR）= 点击数 / 曝光数 |
| **分组** | 流量均分3组（各33.3%） |
| **最小样本** | 每组曝光>3000次（约100次点击） |
| **运行周期** | 7天 |
| **判定标准** | CTR最高且p<0.05的组胜出 |

**案例**：
- A组（黑底产品图）：CTR 2.1%
- B组（男友手部穿搭）：CTR 3.4%
- C组（錾刻微距）：CTR 2.8%
- **结论**：B组胜出，更换为主图

---

### 2.2 标题关键词测试

**测试目标**：找到搜索点击率最高的标题

| 维度 | 设置 |
|------|------|
| **变量** | 标题关键词（A：本命年手链男；B：龙龟银手链；C：错铜工艺手链） |
| **指标** | 搜索点击率、搜索排名变化 |
| **分组** | 同一产品不同标题轮流上架（或AB链接） |
| **运行周期** | 14天（搜索引擎需时间收录） |
| **判定标准** | 搜索CTR提升>20%且排名上升 |

---

### 2.3 价格测试

**测试目标**：找到利润最大化的价格点

| 维度 | 设置 |
|------|------|
| **变量** | 价格（A：468元；B：398元；C：528元） |
| **指标** | 转化率 × 客单价 = 单访客产值 |
| **分组** | 不同SKU或不同时间段（注意库存管理） |
| **运行周期** | 14天 |
| **判定标准** | 单访客产值最高者胜出，非单纯转化率最高 |

**注意**：价格测试易引发用户投诉，建议用"限时活动价"名义测试，测完恢复原价。

---

### 2.4 详情页结构测试

**测试目标**：找到转化率最高的详情页布局

| 维度 | 设置 |
|------|------|
| **变量** | 页面结构（A：卖点→细节→评价→售后；B：视频→卖点→场景→工艺） |
| **指标** | 详情页转化率、页面停留时长、跳失率 |
| **分组** | 使用第三方工具（如AB测试插件）分流 |
| **运行周期** | 14天 |
| **判定标准** | 转化率提升>10%且显著 |

---

### 2.5 投放素材测试

**测试目标**：找到ROI最高的广告素材

| 维度 | 设置 |
|------|------|
| **变量** | 短视频素材（A：工艺过程；B：用户开箱；C：达人推荐） |
| **指标** | ROI = 广告带来的GMV / 广告花费 |
| **分组** | 抖音千川/天猫直通车建立3条计划，预算均分 |
| **运行周期** | 7天，消耗达500元/组后看数据 |
| **判定标准** | ROI>2且p<0.05 |

**优化节奏**：
- 每周上新3-5条素材
- 关停ROI<1的计划
- 对ROI>2的素材加预算复制

---

### 2.6 直播话术测试

**测试目标**：找到转化率最高的直播话术结构

| 维度 | 设置 |
|------|------|
| **变量** | 话术A（价格锚定型）vs 话术B（故事型）vs 话术C（紧迫感型） |
| **指标** | 直播间转化率、停留时长、互动率 |
| **分组** | 同一主播、同一时段、不同场次轮流使用 |
| **运行周期** | 每场直播3小时，测6场（每种2场） |
| **判定标准** | 转化率最高且观众反馈正向 |

---

### 2.7 私域裂变机制测试

**测试目标**：找到拉新成本最低的裂变玩法

| 维度 | 设置 |
|------|------|
| **变量** | A：邀请1人得10元券；B：邀请3人得免费手链；C：拼团2人享8折 |
| **指标** | 拉新成本 = 奖励成本 / 新增用户数 |
| **分组** | 不同用户群推送不同活动 |
| **运行周期** | 7天 |
| **判定标准** | 拉新成本最低且新增用户质量（7日留存率）可接受 |

## 三、样本量计算工具

### 3.1 快速查表法

以点击率测试为例（基准CTR=2%）：

| 预期提升 | 每组最小样本（曝光） | 每组最小点击 |
|----------|---------------------|-------------|
| +20%（到2.4%） | 15,000 | 300 |
| +50%（到3.0%） | 3,000 | 60 |
| +100%（到4.0%） | 1,000 | 20 |

**经验法则**：预期提升越大，需要的样本越小。日常测试建议每组至少3000曝光。

## 五、常见误区与避坑指南

| 误区 | 正确做法 |
|------|----------|
| "A组比B组好10%，立刻切换" | 先算p值，p<0.05才切换 |
| "测试了2天，数据够了" | 至少7天，覆盖完整周期 |
| "同时测5个变量，效率高" | 一次只测1个变量 |
| "测试结果永远有效" | 每季度复测，用户偏好会变 |
| "只测一次就永远用" | 建立测试日历，持续迭代 |

## 六、测试日历建议

| 周次 | 测试内容 | 负责角色 |
|------|----------|----------|
| W1 | 主图测试（新品） | 设计+运营 |
| W2 | 标题关键词测试 | 运营+SEO |
| W3 | 投放素材上新+测试 | 投放+内容 |
| W4 | 详情页结构测试 | 运营+设计 |
| 每月 | 价格敏感度测试 | 运营+供应链 |
| 每季度 | 直播话术迭代测试 | 主播+运营 |

---

> **核心信息**：A/B测试不是玄学，是数学。这套手册给出的是"从假设到决策"的完整工作流——让每一次改版都有数据背书，让每一分钱都花在验证过的方向上。
