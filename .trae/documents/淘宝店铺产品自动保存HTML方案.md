# 淘宝商家店铺产品自动保存为HTML网页 — 实施计划

## 摘要

基于现有爬虫项目（`g:\Downloads\蔡规划\爬虫\`），创建一个**一键自动保存淘宝商家店铺所有产品页面为HTML文件**的Python脚本。目标店铺为"琢匠"，最终保存的HTML文件用于后续数据分析。

## 现状分析

### 已有资源
- **Selenium方案**: `taobao_shop_selenium.py` — 可抓取店铺商品列表（标题/价格/销量/链接），但不保存HTML页面
- **Cookie获取**: `get_cookie.py` / `get_cookie_simple.py` — 可获取登录Cookie
- **HTML解析**: `parse_html_products.py` / `parse_mhtml_batch.py` — 可解析已保存的HTML/MHTML文件
- **JS脚本**: 5个浏览器Console脚本，功能重叠，存在选择器兼容性问题
- **已有数据**: `新建文件夹/` 下有约40个已保存的.mhtml商品详情页

### 核心问题
1. **没有"保存商品详情页HTML"的自动化脚本** — 现有脚本只抓取列表数据，不逐个访问并保存商品详情页
2. **Cookie已过期** — HTTP请求方案无法使用
3. **JS脚本不稳定** — `button:contains()` 等jQuery语法在Console中不兼容
4. **功能分散** — 抓取、保存、解析分散在不同脚本中，需手动串联

## 技术方案

**推荐：Selenium + Chrome 自动化方案**

理由：
- 项目已有Selenium基础设施（`taobao_shop_selenium.py`、`get_cookie.py`）
- 淘宝强制登录+JS渲染，必须用浏览器自动化
- Selenium方案成熟稳定，反检测配置已有参考
- DrissionPage虽更优但引入新依赖，与现有代码风格不一致

## 实施步骤

### 步骤1：创建主脚本 `save_shop_products.py`

**文件**: `g:\Downloads\蔡规划\爬虫\save_shop_products.py`

**功能流程**:
1. 启动Chrome浏览器（复用 `taobao_shop_selenium.py` 的反检测配置）
2. 打开淘宝登录页，等待用户手动登录（或复用已有Cookie）
3. 导航到目标店铺的商品列表页（通过店铺搜索URL）
4. 自动滚动加载所有商品，提取所有商品链接
5. **逐个访问商品详情页**，等待页面完全渲染
6. 将每个商品详情页保存为HTML文件（文件名格式：`{商品标题}_{商品ID}.html`）
7. 保存到指定输出目录（默认 `g:\Downloads\蔡规划\新建文件夹\`）
8. 每次请求间随机延迟（3-8秒），避免触发反爬
9. 记录已保存商品ID，支持断点续传
10. 完成后输出汇总报告（总数/成功/失败/耗时）

**关键实现细节**:
- 使用 `page_source` 获取渲染后的完整HTML
- 文件名清理：去除标题中的特殊字符（`/`、`\`、`:`等）
- 进度显示：当前页码/总页数、当前商品序号
- 异常处理：网络超时重试（最多3次）、跳过无效页面
- 保存进度文件 `saved_products.json`，记录已保存的商品ID和文件路径

### 步骤2：创建配置文件 `config.py`

**文件**: `g:\Downloads\蔡规划\爬虫\config.py`

**内容**:

将配置从主脚本中分离，方便修改目标店铺和参数。

## 假设与决策

| 决策 | 理由 |
|------|------|
| 使用Selenium而非DrissionPage | 与现有代码一致，减少新依赖 |
| 保存HTML而非MHTML | HTML更通用，后续解析工具已支持 |
| 手动登录而非Cookie复用 | Cookie有效期短，手动登录最稳定 |
| 逐个访问详情页而非API | 淘宝API需企业认证，详情页内容最完整 |
| 随机延迟3-8秒 | 平衡速度与反爬风险 |
